## Context

首页是用户登录后的主入口，当前存在以下问题：
1. 快捷服务模块包含hover样式，但H5端触摸屏无hover交互
2. 背景使用静态颜色，视觉效果平淡，且大量元素可能影响渲染性能
3. 重要公告弹窗每次进入首页都展示，影响用户体验

当前技术栈：Vue 3.5 + TDesign Mobile Vue + Tailwind CSS + Pinia

## Goals / Non-Goals

**Goals:**
- 移除H5端无意义的hover样式，减少CSS体积
- 实现流畅美观的背景渐变效果
- 公告弹窗仅在登录后首次展示
- 识别代码库中的重复项，为后续优化提供依据

**Non-Goals:**
- 不重构首页整体架构
- 不修改公告数据结构或后端API
- 不自动删除重复代码（仅标注）

## Decisions

### 1. Hover样式移除方案
**决策**: 直接删除快捷服务模块中的`:hover`相关CSS规则
**理由**: H5触摸屏无hover状态，保留会增加无意义CSS体积
**替代方案**: 使用`:active`替代 - 否决，因为快捷服务点击跳转，无需active反馈

### 2. 背景渐变方案
**决策**: 使用CSS渐变（`linear-gradient`）+ CSS动画实现流动效果
**理由**: 纯CSS实现，无需额外资源加载，性能最优
**替代方案**: 
- Canvas动画 - 否决，性能开销大
- Lottie动画 - 否决，需要引入额外库

```css
/* 渐变方案示意 */
background: linear-gradient(135deg, #667eea 0%, #764ba2 100%);
background-size: 400% 400%;
animation: gradient-flow 15s ease infinite;
```

### 3. 公告展示控制方案
**决策**: 使用`sessionStorage`存储已展示标记
**理由**: 
- 登录后展示一次，关闭浏览器后重置，符合"每次登录"语义
- 无需后端配合，纯前端实现
- sessionStorage随标签页关闭自动清理
**替代方案**:
- localStorage + 登出时清理 - 复杂度高，需监听登出事件
- Pinia状态 - 刷新页面会丢失，不符合需求
- 后端记录已读状态 - 需要API变更，超出范围

### 4. 代码审查方案
**决策**: 人工审查 + 输出报告，不自动修改
**理由**: 重复代码判断需要上下文理解，自动删除风险高

## Risks / Trade-offs

| 风险 | 缓解措施 |
|------|----------|
| 背景动画可能导致低端机卡顿 | 使用`will-change: transform`优化，提供降级方案 |
| sessionStorage在部分浏览器可能被禁用 | 添加try-catch，禁用时不控制展示（每次都弹） |
| 代码审查可能遗漏部分重复项 | 重点关注高频修改模块，输出详细报告供人工复核 |
