# 变更提案：登录页 UI/特效/排版优化

## 变更名称
`optimize-login-page-ui`

## 状态
✅ 已完成

## 背景

### 问题描述
登录页当前存在以下问题：
1. **特效堆叠过多**：粒子（30个DOM）、波浪（3层）、旋转环（3层）、鼠标光晕等特效叠加，真机性能不佳，部分低端机型卡顿
2. **特效不够精美**：动画效果粗糙，缺乏细腻感
3. **布局层次模糊**：Logo、标题、卡片间距不够合理，视觉层次不清晰
4. **字体层级混乱**：标题与副标题差距不够明显，各元素字号不成体系
5. **视觉重心不明确**：Logo 和登录卡片权重不均衡，缺少品牌聚焦

### 影响范围
- 用户首次进入应用的第一印象
- 真机使用体验（流畅度）
- 品牌形象传达

## 动机

作为企业内部办公应用，登录页需要：
- **专业稳重**：不过度炫酷，但要有品质感
- **流畅稳定**：真机体验流畅，不卡顿
- **品牌清晰**：Logo 和品牌名突出，建立信任感
- **操作便捷**：登录卡片醒目，快速进入系统

当前特效虽然丰富，但：
- 粒子用 DOM 实现，性能差
- 波浪 3 层叠加，GPU 合成层多
- 入场动画和逐字动画增加复杂度但价值有限

## 目标

1. 精简特效，提升真机流畅度（DOM 节点减少约 30 个）
2. 优化布局层次，拉开 Logo 和卡片的视觉距离
3. 建立清晰的字体层级系统
4. 平衡 Logo 和卡片的视觉权重，形成上下呼应
5. 保持企业专业风格，突出品牌感

## 非目标

- 不修改适配方案（zoom + vw 体系暂时保持）
- 不修改登录业务逻辑
- 不修改日夜模式切换逻辑
- 不修改品牌色系

### 具体变更

**移除的特效**：
- 登录卡片入场动画（scale + translateY）
- 标题逐字出现动画（v-for + animationDelay）

**精简的特效**：
- 波浪：3层 → 1层
- 粒子：30个 DOM → Canvas 实现

**保留的特效**：
- Logo 旋转环（3层）
- 按钮闪光效果
- 输入框 hover/focus 效果
- 鼠标光晕
- 渐变背景

**布局调整**：
- 副标题与卡片间距：36px → 48px

**字体调整**：
- 主标题：38px → 40px
- 副标题：22px → 20px
- Tab 选项：26px → 28px
- 输入框文字：14px → 16px
- 测试账号提示：统一为 14px

- **回滚**：可通过 Git 回退
