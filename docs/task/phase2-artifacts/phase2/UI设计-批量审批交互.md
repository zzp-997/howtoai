## 三、交互规范

### 3.1 选择交互

- 点击复选框：选中/取消选中，背景色过渡 150ms
- 点击卡片区域（非复选框）：toggle 选中状态
- 全选点击：选中/取消当前页所有可审批项
- 计数动画：数字变化时放大缩小 200ms

### 3.2 批量操作栏显示/隐藏

- 有选中项：底部滑入显示
- 无选中项：向底部滑出隐藏
- 动画：300ms cubic-bezier 弹性效果

### 3.3 批量操作流程

1. 用户选中至少 1 项
2. 底部操作栏滑入，显示选中数量
3. 点击"批量通过"或"批量拒绝"
4. 弹出确认弹窗，显示数量和类型
5. 可选择填写统一意见
6. 点击确认，执行批量操作
7. 显示结果反馈
8. 成功后刷新列表，关闭操作栏

### 3.4 权限控制

- 非部门负责人：隐藏批量操作栏
- 列表项不显示复选框
- 提示文案："您暂无批量审批权限"

## 四、边缘情况处理

### 4.1 超过批量限制

- 选中超过 20 项时：
- 弹出提示："单次最多批量审批 20 项"
- 自动取消超出部分的选中状态

### 4.2 部分项目已处理

- 批量选中包含已审批项目时：
- 过滤掉已审批项，仅对可审批项操作
- 提示："已过滤 X 项已审批项目"

### 4.3 网络错误

- 批量操作失败：
- 显示失败反馈，列出失败项目
- 提供重试按钮

---

## 五、验收要点

1. 复选框选中状态与卡片背景联动，视觉反馈明确
2. 批量操作栏根据选中状态正确显示/隐藏
3. 全选功能正确计算数量
4. 批量限制（20项）有明确提示
5. 确认弹窗信息准确，操作前有二次确认
6. 操作结果反馈清晰，成功/失败状态区分明显
7. 非权限用户无法看到批量操作功能
8. 动画流畅，交互跟手
