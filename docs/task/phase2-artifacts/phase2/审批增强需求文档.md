## 一、概述

### 1.1 功能背景

在企业OA系统中，审批流程是核心功能之一。一期已实现基本的审批功能，但在多级审批、批量操作、审批进度可视化等方面存在不足。本期将对审批功能进行全面增强，提升审批效率和管理水平。

### 1.2 功能目标

- 支持灵活配置多级审批链
- 实现批量审批操作
- 提供审批进度可视化展示
- 增加催办提醒机制
- 完善审批通知体系

## 二、多级审批流程

### 2.1 审批链配置

**功能描述**：管理员可配置不同业务类型的审批链，支持角色级审批、用户级审批、会签审批等多种模式。

**审批节点类型**：
| 类型 | 说明 | 示例 |
|------|------|------|
| role | 按角色审批 | 部门经理、总经理 |
| user | 指定用户审批 | 具体人员 |
| department_head | 部门负责人 | 申请人所在部门负责人 |

**审批模式**：
| 模式 | 说明 |
|------|------|
| or | 或签（任一人审批通过即可进入下一级）|
| and | 会签（所有人审批通过才进入下一级）|

**审批链配置示例**：
```json
{
  "type": "trip",
  "name": "差旅申请审批链",
  "nodes": [
    { "order": 1, "type": "role", "value": "department_manager", "mode": "or" },
    { "order": 2, "type": "user", "value": [1, 2], "mode": "and" },
    { "order": 3, "type": "role", "value": "finance_director", "mode": "or" }
  ]
}
```

### 2.2 审批业务流程

**差旅申请审批流程**：
```
申请人提交 → 部门经理审批 → 财务审批 → 人事备案 → 结束
```

**请假申请审批流程**：
```
申请人提交 → 部门经理审批 → HR审批 → 结束
```

**报销申请审批流程**：
```
申请人提交 → 部门经理审批 → 财务审批 → 出纳付款 → 结束
```

## 三、批量审批

### 3.1 功能描述

支持对待审批列表中的多个申请进行批量审批操作，提高审批效率。

### 3.2 业务规则

| 规则 | 说明 |
|------|------|
| 批量选择 | 最多单次选择20条记录 |
| 批量通过 | 一键通过所有选中申请 |
| 批量拒绝 | 一键拒绝所有选中申请 |
| 差异化审批 | 支持对不同申请填写不同审批意见 |
| 权限控制 | 仅部门负责人及以上角色可使用 |

### 3.3 交互设计

1. 审批列表页提供复选框
2. 底部显示已选数量和操作栏
3. 支持全选当前页/取消全选
4. 批量操作前确认弹窗
5. 操作完成后刷新列表

## 四、审批进度查询

### 4.1 功能描述

申请人可实时查看申请的审批进度，了解当前审批节点和预计审批时间。

### 4.2 进度展示

**进度节点图**：
- 圆形节点表示各审批层级
- 已完成节点显示绿色对勾
- 当前进行中节点显示蓝色动画
- 未开始节点显示灰色
- 被拒绝节点显示红色叉号

**进度详情**：
- 申请人信息
- 申请类型和内容摘要
- 各节点审批人
- 审批意见（如有）
- 审批时间

## 五、催办提醒

### 5.1 催办功能

**功能描述**：申请人可对审批中的申请发起催办，提醒审批人及时处理。

**业务规则**：
| 规则 | 说明 |
|------|------|
| 催办时机 | 申请提交超过24小时未审批 |
| 催办频率 | 同一申请单次迭代最多催办3次 |
| 催办间隔 | 催办后需间隔4小时才能再次催办 |
| 通知方式 | 站内消息 + 邮件（可选）|

### 5.2 自动提醒

**定时任务**：
- 审批人待办超过48小时未处理，系统自动发送提醒
- 每天上午9:00推送当日待办汇总
- 每周五下午推送周待办提醒

## 六、消息通知

### 6.1 通知类型

| 类型 | 触发时机 | 接收人 |
|------|----------|--------|
| 审批通知 | 申请提交、审批通过、审批拒绝、催办 | 申请人、审批人 |
| 系统通知 | 公告发布、系统更新 | 全体用户 |
| 任务通知 | 任务分配、任务评论、截止提醒 | 任务负责人 |

### 6.2 通知渠道

| 渠道 | 说明 |
|------|------|
| 站内消息 | 实时推送，APP消息中心展示 |
| 邮件通知 | 可配置开关，默认开启 |
| 短信通知 | 紧急事项，可配置 |

### 6.3 未读消息角标

- APP底部导航"消息"入口显示未读数量角标
- 角标数字最大显示99+
- 消息已读后角标数字更新
