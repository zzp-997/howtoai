## 一、原型设计概述

本原型设计涵盖审批增强功能的核心页面，包括审批链配置、申请提交、审批详情和进度展示等场景。

---

## 二、页面原型设计

### 2.1 审批链配置页面

**页面定位**：管理员配置各业务类型的审批流程

**核心元素**：
| 元素 | 说明 |
|------|------|
| 顶部标题 | "审批链配置" |
| 业务类型筛选 | 下拉选择：差旅/请假/报销/会议 |
| 审批链列表 | 展示已配置的审批链 |
| 添加按钮 | 新建审批链 |
| 编辑/删除 | 对已有审批链操作 |

**审批链卡片**：
```
┌─────────────────────────────────┐
│ 差旅申请审批链              [编辑][删除] │
│ ─────────────────────────────── │
│ 节点1: 部门经理(角色) - 或签      │
│ 节点2: 财务总监(角色) - 或签      │
│ 节点3: 总经理(角色) - 或签        │
│ 状态: 启用中                      │
└─────────────────────────────────┘
```

**新增/编辑弹窗**：
```
┌─────────────────────────────────┐
│ 配置审批链                       │
│ ─────────────────────────────── │
│ 业务类型: [请选择]              │
│ 审批链名称: [输入框]            │
│                               │
│ 审批节点:                       │
│ ┌─────────────────────────────┐ │
│ │ 节点1: 角色-[部门经理] 模式-[或签] │ │
│ │ 节点2: 用户-[选择用户]    模式-[会签]│ │
│ │ 节点3: 角色-[财务总监] 模式-[或签] │ │
│ └─────────────────────────────┘ │
│ [+ 添加节点]                    │
│                               │
│      [取消]        [保存]       │
└─────────────────────────────────┘
```

### 2.2 审批申请提交页面

**页面定位**：用户提交审批申请

**核心元素**：
| 元素 | 说明 |
|------|------|
| 顶部标题 | "提交审批" |
| 申请类型 | 标签选择：差旅/请假/报销 |
| 申请表单 | 根据类型动态展示 |
| 附件上传 | 支持图片和文档 |
| 预览按钮 | 提交前预览 |
| 提交按钮 | 提交申请 |

**差旅申请表单**：
```
┌─────────────────────────────────┐
│ 提交审批                         │
│ ─────────────────────────────── │
│ 申请类型: [差旅]                │
│                               │
│ 目的地: [输入框]                │
│ 开始日期: [日期选择器]          │
│ 结束日期: [日期选择器]          │
│ 预估费用: [输入框]元           │
│ 申请事由: [多行文本框]         │
│                               │
│ 附件: [上传按钮]               │
│                               │
│ 审批人: 张三(部门经理)         │
│                               │
│      [预览]     [提交]          │
└─────────────────────────────────┘
```

### 2.3 审批列表页面


## 四、意见反馈原型

## 五、交互说明

### 5.1 手势操作
- 左滑列表项：显示删除按钮
- 下拉列表：刷新数据
- 上拉加载：加载更多

### 5.2 动画效果
- 页面切换：右滑进入，左滑退出
- 状态变更：渐变过渡
- 按钮点击：涟漪效果
- 加载状态：骨架屏

---

## 六、验收要点

1. 所有页面布局符合移动端规范
2. 交互流程清晰，用户操作路径短
3. 状态变化有明确视觉反馈
4. 空数据场景有友好提示
5. 错误场景有清晰引导
