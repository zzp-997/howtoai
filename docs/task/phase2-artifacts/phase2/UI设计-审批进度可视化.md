```

- 已完成段：2px solid #10B981
- 进行中段：2px solid #3B82F6，虚线
```

## 三、交互规范

### 3.1 页面加载动画

- 顶部卡片：渐入 + 轻微上滑，300ms
- 进度节点：依次从左到右淡入，每个 delay 150ms
- 连接线：从左到右绘制动画
- 申请信息：淡入，delay 400ms
- 审批记录：依次淡入，delay 500ms，每个 100ms

### 3.2 节点交互

- 点击节点：显示节点详情弹窗
- 节点悬停：轻微放大 1.1x

### 3.3 催办交互

- 点击催办：
  1. 按钮显示 loading 状态
  2. 发送催办请求
  3. 显示成功/失败反馈
  4. 更新催办次数显示

### 3.4 下拉刷新

- 刷新审批进度数据
- 显示刷新动画
- 更新节点状态

## 四、边缘情况处理

### 4.1 审批超时

- 节点停留超过 48 小时：
- 节点显示警告图标 ⚠️
- 显示"等待时间较长"提示

### 4.2 审批链异常

- 审批人离职/不存在：
- 显示"待定"状态
- 提供"重新指定审批人"入口（管理员）

### 4.3 移动端适配

- 节点图改为纵向排列
- 节点自动换行显示

---

## 五、验收要点

1. 进度节点图正确展示各节点状态（已完成/进行中/待处理/已拒绝）
2. 节点状态颜色区分明显
3. 进行中节点有脉冲动画效果
4. 连接线正确显示进度
5. 审批记录按时间顺序展示
6. 当前审批节点突出显示
7. 催办按钮根据业务规则正确启用/禁用
8. 页面加载动画流畅
9. 移动端适配良好
