## 一、概述

### 1.1 背景

知识库模块旨在为企业提供统一的文档管理和知识共享平台，帮助员工快速查找所需资料，促进企业内部知识的积累和传播。

### 1.2 目标

- 提供统一的文档管理平台
- 支持多级分类体系
- 实现全文检索功能
- 控制文档访问权限
- 追踪文档阅读情况

---

## 二、功能需求

### 2.1 知识分类管理

#### 2.1.1 功能描述

支持创建多级分类目录，便于文档的组织和管理。

#### 2.1.2 分类结构

| 字段 | 说明 |
|------|------|
| 分类名称 | 分类的显示名称 |
| 父级分类 | 支持多级嵌套（最多3级） |
| 排序权重 | 数字越小越靠前 |
| 创建时间 | 分类创建时间 |

#### 2.1.3 分类操作

- 新增分类
- 编辑分类名称和排序
- 删除分类（需确认无关联文档）
- 分类拖拽排序

---

### 2.2 知识文档管理

#### 2.2.1 功能描述

支持文档的创建、编辑、发布和删除操作。

#### 2.2.2 文档属性

| 字段 | 说明 |
|------|------|
| 标题 | 文档标题，必填，最大200字符 |
| 内容 | 文档正文，支持富文本 |
| 分类 | 关联的分类 |
| 标签 | 自定义标签，支持多选 |
| 可见范围 | public/department/custom |
| 状态 | draft/published |

#### 2.2.3 文档操作

- 新建文档
- 编辑文档
- 发布/取消发布
- 删除文档（软删除）
- 复制文档
- 移动文档到其他分类

---

### 2.3 全文检索

#### 2.3.1 功能描述

支持对文档标题和内容进行全文搜索。

#### 2.3.2 搜索功能

| 功能 | 说明 |
|------|------|
| 关键词搜索 | 支持标题和内容关键词匹配 |
| 分类筛选 | 在指定分类内搜索 |
| 标签筛选 | 按标签筛选文档 |
| 时间范围 | 按发布时间筛选 |
| 排序方式 | 相关度/发布时间/阅读量 |

#### 2.3.3 搜索结果

- 高亮显示匹配的关键词
- 显示文档标题、分类、摘要
- 支持搜索结果分页

---

### 2.4 权限控制

#### 2.4.1 可见范围

| 范围 | 说明 |
|------|------|
| 公开(public) | 所有用户可查看 |
| 部门(department) | 仅本部门用户可查看 |
| 自定义(custom) | 指定用户或角色可查看 |

#### 2.4.2 管理权限

| 角色 | 权限 |
|------|------|
| 普通员工 | 查看公开文档、创建个人文档 |
| 部门经理 | 管理本部门文档 |
| 知识管理员 | 管理所有分类和文档 |
| 管理员 | 全部权限 |

---

### 2.5 阅读统计

#### 2.5.1 功能描述

追踪文档的阅读情况，提供热门文档推荐。

#### 2.5.2 统计指标

| 指标 | 说明 |
|------|------|
| 阅读次数 | 文档被查看的总次数 |
| 阅读人数 | 阅读文档的唯一用户数 |
| 最后阅读时间 | 最近一次阅读的时间 |

#### 2.5.3 热门文档

- 按阅读量排序的推荐列表
- 近期更新的文档列表

## 三、数据设计

### 3.1 知识分类表

```sql
CREATE TABLE knowledge_categories (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  name VARCHAR(100) NOT NULL COMMENT '分类名称',
  parent_id BIGINT NULL COMMENT '父级分类ID',
  sort_order INT DEFAULT 0 COMMENT '排序权重',
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_parent (parent_id),
  INDEX idx_sort (sort_order)
);
```

### 3.2 知识内容表

```sql
CREATE TABLE knowledges (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  title VARCHAR(200) NOT NULL COMMENT '文档标题',
  content TEXT NOT NULL COMMENT '文档内容',
  category_id BIGINT COMMENT '分类ID',
  author_id BIGINT NOT NULL COMMENT '作者ID',
  tags JSON COMMENT '标签JSON数组',
  visibility ENUM('public', 'department', 'custom') DEFAULT 'public' COMMENT '可见范围',
  custom_users JSON COMMENT '自定义可见用户',
  view_count INT DEFAULT 0 COMMENT '阅读次数',
  status ENUM('draft', 'published') DEFAULT 'draft' COMMENT '状态',
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  deleted_at DATETIME NULL COMMENT '删除时间',
  INDEX idx_category (category_id),
  INDEX idx_author (author_id),
  INDEX idx_status (status),
  FULLTEXT INDEX ft_title_content (title, content)
);
```

## 四、交互设计

### 4.1 列表页交互

- 分类树形展示，支持展开/折叠
- 文档列表支持卡片视图和列表视图切换
- 批量操作：移动分类、删除、发布

### 4.2 编辑页交互

- 富文本编辑器：支持文字格式、图片上传、表格
- 草稿自动保存（每30秒）
- 预览功能

### 4.3 搜索交互

- 搜索框即时联想
- 搜索结果高亮关键词
- 搜索历史记录
