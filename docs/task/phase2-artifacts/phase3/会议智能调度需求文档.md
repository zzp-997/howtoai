## 一、概述

### 1.1 背景

会议智能调度模块旨在优化会议室预约流程，通过智能推荐算法帮助用户快速找到合适的会议室和时间，减少会议室冲突，提升会议效率。

### 1.2 目标

- 智能推荐最佳会议室和时间
- 实时检测会议室冲突
- 提供会议室可视化视图
- 优化会议室资源分配

---

## 二、功能需求

### 2.1 智能推荐

#### 2.1.1 功能描述

根据用户输入的会议需求，智能推荐最合适的会议室和时间。

#### 2.1.2 推荐条件

| 输入条件 | 说明 |
|----------|------|
| 参会人数 | 必填，用于匹配容量合适的会议室 |
| 会议时长 | 必填，用于计算可用时段 |
| 期望日期 | 必填，支持多选 |
| 期望时段 | 可选，如上午/下午/指定时间段 |
| 会议设备 | 可选，如投影仪、视频会议设备、白板 |
| 楼层偏好 | 可选，优先推荐指定楼层的会议室 |

#### 2.1.3 推荐逻辑

```
优先级排序：
1. 优先匹配容量 >= 参会人数的会议室
2. 优先选择满足设备需求的会议室
3. 优先选择距离用户所在楼层最近的会议室
4. 优先推荐利用率较低的会议室
5. 时间段按利用率从低到高排序
```

#### 2.1.4 推荐结果展示

- 显示推荐的会议室列表
- 显示各会议室的可用时段
- 显示推荐原因（如：空闲最多、设备匹配、距离最近）

---

### 2.2 会议室冲突检测

#### 2.2.1 功能描述

在预约时实时检测时间冲突，避免重复预订。

#### 2.2.2 冲突规则

| 场景 | 处理 |
|------|------|
| 完全重叠 | 提示冲突，建议其他时间 |
| 部分重叠 | 提示冲突时间段 |
| 边界时间 | 支持首尾相接（上一会议结束=下一会议开始） |

#### 2.2.2 冲突提示

- 实时检测，冲突时阻止提交
- 显示冲突的会议信息（时间、预订人）
- 提供一键查看替代方案

---

### 2.3 会议室可视化

#### 2.3.1 功能描述

以日历/时间轴形式展示会议室预约情况。

#### 2.3.2 展示形式

| 视图 | 说明 |
|------|------|
| 日视图 | 展示当天各会议室的时间占用 |
| 周视图 | 展示一周内会议室的预约情况 |
| 会议室详情 | 展示单个会议室的预约日历 |

#### 2.3.3 可视化元素

- 已预约：显示会议主题和预订人
- 可预约：显示空余时段
- 我的预约：特殊高亮显示当前用户的预约
- 冲突提示：红色标记冲突时段

---

### 2.4 自动排程优化

#### 2.4.1 功能描述

当多个会议需要协调时，提供自动排程建议。

#### 2.4.2 排程场景

| 场景 | 说明 |
|------|------|
| 周期性会议 | 每周/每月固定时间的会议，自动寻找最佳时段 |
| 大型会议 | 多人参会会议，寻找容量足够的空闲会议室 |
| 跨部门会议 | 协调多方时间，寻找共同空闲时段 |

#### 2.4.3 优化建议

- 显示优化后的时间方案
- 显示各参与者的空闲时间叠加
- 一键确认排程建议

## 三、数据设计

### 3.1 会议室基础信息表

```sql
CREATE TABLE meeting_rooms (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  name VARCHAR(100) NOT NULL COMMENT '会议室名称',
  floor VARCHAR(50) COMMENT '楼层',
  capacity INT DEFAULT 0 COMMENT '容纳人数',
  equipment JSON COMMENT '设备列表',
  status ENUM('normal', 'maintenance', 'disabled') DEFAULT 'normal' COMMENT '状态',
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_floor (floor),
  INDEX idx_capacity (capacity),
  INDEX idx_status (status)
);
```

### 3.2 会议调度历史表

```sql
CREATE TABLE meeting_schedules (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  title VARCHAR(200) NOT NULL COMMENT '会议标题',
  room_id BIGINT NOT NULL COMMENT '会议室ID',
  organizer_id BIGINT NOT NULL COMMENT '组织者ID',
  attendees JSON COMMENT '参会人员ID列表',
  start_time DATETIME NOT NULL COMMENT '开始时间',
  end_time DATETIME NOT NULL COMMENT '结束时间',
  recurrence_rule VARCHAR(100) COMMENT '重复规则',
  status ENUM('scheduled', 'cancelled', 'completed') DEFAULT 'scheduled' COMMENT '状态',
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_room_time (room_id, start_time, end_time),
  INDEX idx_organizer (organizer_id),
  INDEX idx_status (status)
);
```

### 3.3 智能推荐日志表

```sql
CREATE TABLE schedule_recommendations (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  user_id BIGINT NOT NULL COMMENT '用户ID',
  criteria JSON NOT NULL COMMENT '推荐条件',
  recommendations JSON NOT NULL COMMENT '推荐结果',
  selected_room_id BIGINT COMMENT '用户选择的会议室',
  selected_time DATETIME COMMENT '用户选择的时间',
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_user (user_id)
);
```

## 四、算法设计

### 4.1 推荐算法

```python
def recommend_room(attendees_count, duration, date, required_equipment):
    # 1. 筛选满足容量的会议室
    available_rooms = rooms.filter(capacity >= attendees_count)

    # 2. 筛选满足设备的会议室
    if required_equipment:
        available_rooms = rooms.filter(has_all_equipment(required_equipment))

    # 3. 计算各会议室的空闲时段
    for room in available_rooms:
        free_slots = calculate_free_slots(room, date, duration)
        room.score = calculate_score(room, free_slots)

    # 4. 按得分排序返回推荐结果
    return sort_by_score(available_rooms)
```

### 4.2 冲突检测算法

```python
def check_conflict(room_id, start_time, end_time):
    # 查询该会议室在指定时间范围内的预约
    existing = schedules.filter(
        room_id=room_id,
        status='scheduled',
        start_time__lt=end_time,
        end_time__gt=start_time
    )
    return existing if existing else None
```

## 五、交互设计

### 5.1 预约流程

```
1. 填写会议信息（标题、人数、时长）
2. 选择日期和时段
3. 系统智能推荐会议室
4. 用户选择或手动选择会议室
5. 添加参会人员（可选）
6. 确认预约
```

### 5.2 智能推荐交互

- 输入条件后实时显示推荐结果
- 支持手动刷新推荐
- 显示推荐原因（帮助用户理解）
- 支持按条件过滤推荐结果

## 六、验收标准

### 6.1 功能验收

| 验收项 | 验收标准 |
|--------|----------|
| 智能推荐 | 输入条件后推荐结果准确，排序合理 |
| 冲突检测 | 实时检测冲突，提示清晰 |
| 可视化 | 日/周视图展示正确，可交互 |
| 自动排程 | 排程建议合理，可一键确认 |
| 会议室管理 | 增删改查正常，状态切换正常 |

### 6.2 性能要求

| 指标 | 标准 |
|------|------|
| 推荐响应 | < 1秒 |
| 冲突检测 | < 200ms |
| 日视图加载 | < 2秒 |
| 周视图加载 | < 3秒 |

### 6.3 异常处理

| 场景 | 处理 |
|------|------|
| 无可用会议室 | 提示建议更换时间或日期 |
| 推荐结果全被占用 | 提示手动选择并显示冲突时间 |
| 会议室维护 | 维护期间不显示该会议室 |
