### 2.2 安全性分析 (82/100)

#### 2.2.1 密码存储安全

| 评估项 | 当前方案 | 安全评级 | 建议 |
|--------|----------|----------|------|
| 哈希算法 | bcrypt（建议） | 优秀 | 确认使用 bcrypt 或 Argon2，避免使用 MD5/SHA1 |
| 盐值处理 | bcrypt 内置 | 优秀 | 无需额外处理 |
| 历史密码存储 | 哈希存储 | 良好 | 确认每次密码变更都使用新的盐值 |

**关键建议**：
- 在需求文档中明确 bcrypt 的 work factor 参数（建议 >= 12）
- 密码历史表应只存储哈希值，不存储盐值（bcrypt 已内置）

#### 2.2.2 Token 机制安全

| 评估项 | 当前方案 | 安全评级 | 建议 |
|--------|----------|----------|------|
| 双 Token 机制 | Access + Refresh | 优秀 | 符合最佳实践 |
| Token 有效期 | 2h / 7d | 良好 | 建议根据安全等级可配置 |
| Token 黑名单 | Redis 存储 | 良好 | 需确保 Redis 持久化配置正确 |
| Refresh Token 轮换 | 支持 | 优秀 | 防止 Token 重放攻击 |
| 签名算法 | HS256/RS256 | 良好 | 生产环境建议使用 RS256 |

**关键问题**：

**问题 A：Token 黑名单缺失 TTL 管理**
- 风险：黑名单数据可能无限增长
- 建议：为黑名单记录设置与 Token 过期时间相同的 TTL


**问题 B：缺失 Token 签名密钥轮换机制**
- 风险：密钥泄露后无法快速响应
- 建议：实现密钥轮换机制，支持多密钥并存的过渡期

**性能问题 D：高频登录查询优化**

当前失败计数查询：
```sql
SELECT COUNT(*) FROM login_logs
WHERE user_id = ? AND status = 'failed'
  AND created_at >= DATE_SUB(NOW(), INTERVAL 30 MINUTE);
```

**问题**：每次登录都需执行 COUNT 聚合查询，数据量大时性能下降。

**建议**：使用 Redis 缓存失败计数

```python
# 使用 Redis INCR 实现计数
fail_count = redis.incr(f"login_fail:{user_id}")
if fail_count == 1:
    redis.expire(f"login_fail:{user_id}", 1800)  # 30分钟过期

# 登录成功时重置
redis.delete(f"login_fail:{user_id}")
```

#### 2.3.2 日志写入性能

**问题 E：日志同步写入影响主流程性能**

### 4.3 代码实现建议

#### 4.3.1 登录失败计数（使用 Redis）

```python
class LoginAttemptService:
    def __init__(self, redis_client):
        self.redis = redis_client
        self.max_attempts = 5
        self.lock_duration = 900  # 15分钟
        self.count_ttl = 1800     # 30分钟

    def record_failure(self, user_id: int) -> dict:
        """记录登录失败"""
        key = f"login_fail:{user_id}"
        lock_key = f"login_lock:{user_id}"

        # 检查是否已锁定
        if self.redis.exists(lock_key):
            ttl = self.redis.ttl(lock_key)
            return {"locked": True, "remain_seconds": ttl}

        # 增加失败计数
        count = self.redis.incr(key)
        if count == 1:
            self.redis.expire(key, self.count_ttl)

        # 达到阈值则锁定
        if count >= self.max_attempts:
            self.redis.setex(lock_key, self.lock_duration, "1")
            self.redis.delete(key)
            return {"locked": True, "remain_seconds": self.lock_duration}

        return {"locked": False, "remain_attempts": self.max_attempts - count}

    def reset_failure(self, user_id: int):
        """登录成功后重置"""
        self.redis.delete(f"login_fail:{user_id}")
        self.redis.delete(f"login_lock:{user_id}")
```

#### 4.3.2 Token 黑名单管理

```python
class TokenBlacklistService:
    def __init__(self, redis_client, refresh_expire_seconds: int):
        self.redis = redis_client
        self.default_ttl = refresh_expire_seconds

    def add_to_blacklist(self, token_jti: str, ttl: int = None):
        """将 Token 加入黑名单"""
        ttl = ttl or self.default_ttl
        self.redis.setex(f"token_blacklist:{token_jti}", ttl, "1")

    def is_blacklisted(self, token_jti: str) -> bool:
        """检查 Token 是否在黑名单中"""
        return self.redis.exists(f"token_blacklist:{token_jti}")
```
