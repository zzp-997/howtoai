## 一、设计概述

### 1.1 设计目标

本文档为「极智协同」企业OA办公应用二期迭代第一阶段的安全模块设计数据库表结构，主要支持以下安全功能：

1. **登录失败限制** - 记录登录日志，支持失败计数查询，实现账户锁定机制
2. **操作日志审计** - 记录用户关键操作，支持安全审计和问题追溯
3. **密码安全策略** - 支持密码历史检测、过期提醒等机制

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| 性能优先 | 为高频查询设计合适的索引，避免全表扫描 |
| 数据完整性 | 使用合适的数据类型和约束，确保数据准确性 |
| 安全性 | 敏感数据加密存储，日志脱敏处理 |
| 可扩展性 | 预留扩展字段，支持未来功能迭代 |
| 运维友好 | 设计合理的索引命名规范，便于监控和优化 |

### 1.3 表清单

| 表名 | 用途 | 状态 |
|------|------|------|
| login_logs | 登录日志表 | 新增 |
| operation_logs | 操作日志表 | 新增 |
| password_history | 密码历史表 | 新增 |
| users | 用户表扩展 | 修改 |

## 二、表结构设计

### 2.1 登录日志表 (login_logs)

#### 2.1.1 DDL 语句

```sql
-- 登录日志表
CREATE TABLE login_logs (
    id BIGINT UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '主键ID',
    user_id BIGINT UNSIGNED NULL COMMENT '用户ID，用户不存在时为NULL',
    username VARCHAR(100) NOT NULL COMMENT '尝试登录的用户名',
    ip_address VARCHAR(45) NOT NULL COMMENT '登录IP地址（支持IPv6）',
    device_info VARCHAR(500) NULL COMMENT '设备信息（UA、设备类型等）',
    status ENUM('success', 'failed') NOT NULL COMMENT '登录状态：success-成功, failed-失败',
    failure_reason VARCHAR(50) NULL COMMENT '失败原因：password_error/user_disabled/account_locked/user_not_found',
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '登录时间',

    PRIMARY KEY (id),
    -- 用户维度查询索引：用于登录失败计数、用户登录历史查询
    INDEX idx_user_status_time (user_id, status, created_at),
    -- 用户名维度查询索引：用于用户名维度的登录失败计数
    INDEX idx_username_time (username, created_at),
    -- 时间维度索引：用于日志归档清理
    INDEX idx_created_time (created_at),
    -- IP维度索引：用于IP维度的安全分析
    INDEX idx_ip_time (ip_address, created_at)

) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='登录日志表';
```

#### 2.1.2 字段说明

| 字段名 | 类型 | 是否必填 | 默认值 | 说明 |
|--------|------|----------|--------|------|
| id | BIGINT UNSIGNED | 是 | AUTO_INCREMENT | 主键，自增 |
| user_id | BIGINT UNSIGNED | 否 | NULL | 用户ID，用户不存在时为NULL（防止枚举攻击） |
| username | VARCHAR(100) | 是 | - | 尝试登录的用户名，用于失败计数 |
| ip_address | VARCHAR(45) | 是 | - | 登录IP地址，支持IPv6格式 |
| device_info | VARCHAR(500) | 否 | NULL | 设备信息，包含UA、设备类型、操作系统等 |
| status | ENUM | 是 | - | 登录状态：success/failed |
| failure_reason | VARCHAR(50) | 否 | NULL | 失败原因枚举值 |
| created_at | DATETIME | 是 | CURRENT_TIMESTAMP | 登录时间，UTC存储 |

#### 2.1.3 failure_reason 枚举值

| 枚举值 | 说明 |
|--------|------|
| password_error | 密码错误 |
| user_disabled | 用户已被禁用 |
| account_locked | 账户已锁定 |
| user_not_found | 用户不存在（内部标识，不暴露给前端） |

#### 2.1.4 索引设计说明

| 索引名 | 索引类型 | 字段 | 用途说明 |
|--------|----------|------|----------|
| PRIMARY | 主键 | id | 唯一标识 |
| idx_user_status_time | 组合索引 | user_id, status, created_at | 查询用户登录失败次数（核心查询） |
| idx_username_time | 组合索引 | username, created_at | 按用户名查询最近的登录尝试 |
| idx_created_time | 普通索引 | created_at | 日志归档清理，按时间范围删除 |
| idx_ip_time | 组合索引 | ip_address, created_at | IP维度的安全分析 |

**查询示例**：

```sql
-- 查询用户最近30分钟内的登录失败次数（核心查询）
SELECT COUNT(*) AS fail_count
FROM login_logs
WHERE user_id = ?
  AND status = 'failed'
  AND created_at >= DATE_SUB(NOW(), INTERVAL 30 MINUTE);

-- 执行计划分析（预期使用 idx_user_status_time 索引）
EXPLAIN SELECT COUNT(*) FROM login_logs 
WHERE user_id = 123 AND status = 'failed' AND created_at >= DATE_SUB(NOW(), INTERVAL 30 MINUTE);
-- type: ref, key: idx_user_status_time, rows: 预估行数
```

### 2.2 操作日志表 (operation_logs)

#### 2.2.1 DDL 语句

```sql
-- 操作日志表
CREATE TABLE operation_logs (
    id BIGINT UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '主键ID',
    user_id BIGINT UNSIGNED NOT NULL COMMENT '操作用户ID',
    username VARCHAR(100) NOT NULL COMMENT '操作用户名（冗余存储，防止用户删除后日志失效）',
    action VARCHAR(50) NOT NULL COMMENT '操作类型：login/logout/create/update/delete/approve/reject等',
    module VARCHAR(50) NOT NULL COMMENT '操作模块：auth/attendance/approval/meeting/trip等',
    resource_type VARCHAR(50) NULL COMMENT '资源类型：user/meeting/trip/leave等',
    resource_id BIGINT UNSIGNED NULL COMMENT '资源ID',
    detail JSON NULL COMMENT '操作详情，JSON格式存储变更前后数据',
    ip_address VARCHAR(45) NOT NULL COMMENT '客户端IP地址',
    user_agent VARCHAR(500) NULL COMMENT '客户端UA信息',
    request_id VARCHAR(50) NULL COMMENT '请求唯一标识（用于追踪）',
    status ENUM('success', 'failed') NOT NULL DEFAULT 'success' COMMENT '操作结果',
    error_message TEXT NULL COMMENT '失败时的错误信息',
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '操作时间',

    PRIMARY KEY (id),
    -- 用户维度查询索引：用于查询某用户的操作历史
    INDEX idx_user_action_time (user_id, action, created_at),
    -- 模块维度查询索引：用于按模块筛选日志
    INDEX idx_module_time (module, created_at),
    -- 资源维度查询索引：用于查询某资源的所有操作记录
    INDEX idx_resource (resource_type, resource_id),
    -- 时间维度索引：用于日志归档清理
    INDEX idx_created_time (created_at),
    -- 请求ID索引：用于请求追踪
    INDEX idx_request_id (request_id)

) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='操作日志表';
```

### 2.3 密码历史表 (password_history)

#### 2.3.1 DDL 语句

```sql
-- 密码历史记录表
CREATE TABLE password_history (
    id BIGINT UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '主键ID',
    user_id BIGINT UNSIGNED NOT NULL COMMENT '用户ID',
    password_hash VARCHAR(255) NOT NULL COMMENT '密码哈希值',
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',

    PRIMARY KEY (id),
    -- 用户维度索引：查询用户密码历史
    INDEX idx_user_created (user_id, created_at)

) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='密码历史记录表';
```

#### 2.3.2 字段说明

| 字段名 | 类型 | 是否必填 | 默认值 | 说明 |
|--------|------|----------|--------|------|
| id | BIGINT UNSIGNED | 是 | AUTO_INCREMENT | 主键，自增 |
| user_id | BIGINT UNSIGNED | 是 | - | 用户ID |
| password_hash | VARCHAR(255) | 是 | - | 密码哈希值（使用bcrypt等安全算法） |
| created_at | DATETIME | 是 | CURRENT_TIMESTAMP | 密码修改时间 |

#### 2.3.3 设计说明

密码历史表用于存储用户最近使用的密码哈希值，防止用户重复使用旧密码。

**查询示例**：

```sql
-- 检查新密码是否在最近5次历史密码中
SELECT COUNT(*) > 0 AS is_duplicate
FROM (
    SELECT password_hash
    FROM password_history
    WHERE user_id = ?
    ORDER BY created_at DESC
    LIMIT 5
) AS recent_history
WHERE password_hash = ?;

-- 维护：只保留最近5个历史记录
DELETE FROM password_history
WHERE user_id = ?
  AND id NOT IN (
    SELECT id FROM (
      SELECT id FROM password_history
      WHERE user_id = ?
      ORDER BY created_at DESC
      LIMIT 5
    ) AS recent
  );
```

```sql
ALTER TABLE users
```

#### 2.4.2 扩展字段说明

| 字段名 | 类型 | 是否必填 | 默认值 | 说明 |
|--------|------|----------|--------|------|
| login_fail_count | INT UNSIGNED | 是 | 0 | 连续登录失败次数，登录成功后重置为0 |
| locked_until | DATETIME | 否 | NULL | 账户锁定截止时间，NULL表示未锁定 |
| last_login_attempt | DATETIME | 否 | NULL | 最近一次登录尝试时间，用于计数有效期判断 |
| password_changed_at | DATETIME | 否 | NULL | 密码最后修改时间，用于密码过期检测 |
| password_expire_days | INT UNSIGNED | 是 | 90 | 密码有效期（天），0表示永不过期 |
| require_password_change | TINYINT(1) | 是 | 0 | 是否需要强制修改密码 |

#### 2.4.3 业务逻辑说明

**登录失败限制逻辑**：

```sql
-- 1. 检查账户是否锁定
SELECT id, login_fail_count, locked_until, last_login_attempt
FROM users
WHERE username = ?;

-- 2. 登录失败时更新失败计数
UPDATE users
SET login_fail_count = login_fail_count + 1,
    last_login_attempt = NOW(),
    locked_until = CASE 
        WHEN login_fail_count + 1 >= 5 THEN DATE_ADD(NOW(), INTERVAL 15 MINUTE)
        ELSE locked_until
    END
WHERE id = ?;

-- 3. 登录成功时重置失败计数
UPDATE users
SET login_fail_count = 0,
    locked_until = NULL,
    last_login_attempt = NOW()
WHERE id = ?;

-- 4. 定时清理过期的锁定状态（可选，在登录时检查更准确）
UPDATE users
SET locked_until = NULL
WHERE locked_until IS NOT NULL AND locked_until < NOW();
```

**密码过期检测逻辑**：

```sql
-- 查询密码即将过期的用户（提前7天提醒）
SELECT id, username, password_changed_at, password_expire_days
FROM users
WHERE password_expire_days > 0
  AND password_changed_at IS NOT NULL
  AND DATE_ADD(password_changed_at, INTERVAL password_expire_days DAY) BETWEEN NOW() AND DATE_ADD(NOW(), INTERVAL 7 DAY);

-- 查询密码已过期的用户
SELECT id, username
FROM users
WHERE password_expire_days > 0
  AND password_changed_at IS NOT NULL
  AND DATE_ADD(password_changed_at, INTERVAL password_expire_days DAY) < NOW();
```

```sql
CREATE PROCEDURE add_index_if_not_exists()
BEGIN
    -- 添加 idx_locked_until 索引
    IF NOT EXISTS (
        SELECT 1 FROM information_schema.statistics 
        WHERE table_schema = DATABASE() 
          AND table_name = 'users' 
          AND index_name = 'idx_locked_until'
    ) THEN
        ALTER TABLE users ADD INDEX idx_locked_until (locked_until);
    END IF;
    
    -- 添加 idx_password_changed 索引
    IF NOT EXISTS (
        SELECT 1 FROM information_schema.statistics 
        WHERE table_schema = DATABASE() 
          AND table_name = 'users' 
          AND index_name = 'idx_password_changed'
    ) THEN
        ALTER TABLE users ADD INDEX idx_password_changed (password_changed_at);
    END IF;
END //
DELIMITER ;

CALL add_index_if_not_exists();
DROP PROCEDURE IF EXISTS add_index_if_not_exists;

-- 初始化密码修改时间
UPDATE users
SET password_changed_at = NOW()
WHERE password_changed_at IS NULL;

COMMIT;

-- 回滚脚本文件名: V20260430_security_module_rollback.sql
```

### 3.5 数据归档策略

考虑到日志表数据量增长较快，建议制定归档策略：

```sql
-- 创建归档表（按月分表）
CREATE TABLE operation_logs_archive_202604 LIKE operation_logs;
CREATE TABLE login_logs_archive_202604 LIKE login_logs;
```

```sql
SELECT COUNT(*) AS fail_count
```

## 五、安全建议

### 5.1 敏感数据处理

| 数据类型 | 处理方式 |
|----------|----------|
| 密码哈希 | 使用bcrypt/Argon2，存储在password_history表 |
