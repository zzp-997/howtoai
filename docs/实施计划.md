# 企业办公助手 H5 端实施方案

## 一、项目概述

### 1.1 项目目标
将现有 "How To AI" 学习平台改造为企业办公助手 H5 端，实现 6 大核心功能模块，数据完全存储于 IndexedDB，支持离线运行。

### 1.2 技术栈确认
- **前端框架**：Vue 3.5 + Vue Router 4 + Pinia 2
- **UI 框架**：TDesign Mobile Vue 1.6
- **样式方案**：TailwindCSS 3.4 + Less
- **数据存储**：IndexedDB（浏览器本地）
- **工具库**：dayjs（日期处理）、lodash（工具函数）

### 1.3 关键设计决策
| 决策项 | 选择 | 说明 |
|--------|------|------|
| 预定日历展示 | 时间轴列表 | 选定日期后按时间段展示预定情况 |
| 补卡审批流程 | 独立审批流程 | 补卡单作为独立申请，走用户发起→管理员审批 |
| 数据库初始化 | 应用启动时自动初始化 | main.js 入口处检测并初始化 |
| 首页设计 | 功能入口卡片式 | 展示各功能模块入口卡片 |

## 四、关键技术实现要点

### 4.1 IndexedDB 封装示例

```javascript
// src/db/index.js
const DB_NAME = 'OfficeAssistantDB';
const DB_VERSION = 1;

let db = null;

export async function initDB() {
  return new Promise((resolve, reject) => {
    const request = indexedDB.open(DB_NAME, DB_VERSION);

    request.onerror = () => reject(request.error);
    request.onsuccess = () => {
      db = request.result;
      resolve(db);
    };

    request.onupgradeneeded = (event) => {
      const database = event.target.result;
      // 创建各表
      createStores(database);
    };
  });
}

function createStores(db) {
  // users 表
  if (!db.objectStoreNames.contains('users')) {
    const userStore = db.createObjectStore('users', { keyPath: 'id', autoIncrement: true });
    userStore.createIndex('username', 'username', { unique: true });
  }
  // ... 其他表
}
```

### 4.2 登录改造要点

```javascript
// 登录验证逻辑改为 IndexedDB 查询
import { getUserByUsername } from '@/db/repositories/userRepo';

const handleLogin = async () => {
  const user = await getUserByUsername(formData.username);
  if (!user || user.password !== formData.password) {
    throw new Error('用户名或密码错误');
  }
  if (selectedRole !== user.role) {
    throw new Error('角色不匹配');
  }
  // 存储用户信息到 Pinia
  userStore.setUser(user);
  // 根据角色跳转
  router.push(selectedRole === 'admin' ? '/admin/meeting' : '/user/meeting');
};
```

### 4.3 文件存储方案

```javascript
// 文档上传时转换为 Blob 存储
async function uploadDocument(file, categoryId, tags) {
  const reader = new FileReader();
  reader.onload = async () => {
    const blob = new Blob([reader.result], { type: file.type });
    await documentRepo.create({
      name: file.name,
      categoryId,
      tags,
      fileBlob: blob,
      fileType: file.type,
      fileSize: file.size,
      uploadBy: userStore.userInfo.id,
      uploadAt: new Date()
    });
  };
  reader.readAsArrayBuffer(file);
}

// 文档预览/下载
function previewDocument(doc) {
  const url = URL.createObjectURL(doc.fileBlob);
  window.open(url);
}
```

## 五、风险与应对

| 风险项 | 影响 | 应对措施 |
|--------|------|----------|
| IndexedDB 存储限制 | 大文件存储可能超限 | 限制单文件 ≤20MB，提示用户定期导出备份 |
| 浏览器兼容性 | 部分浏览器可能不支持 | 使用标准 API，添加兼容性检测提示 |
| 数据迁移 | 后续转服务端架构 | 设计 Repository 层，替换为 API 调用即可 |
