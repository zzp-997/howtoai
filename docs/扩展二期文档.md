### 1. 待办事项 / 任务管理
职场痛点
员工每天面临大量零散任务（回复邮件、提交报告、跟进客户等），缺乏统一记录和提醒工具，容易遗漏。

功能描述
提供一个轻量级个人待办清单，支持创建、编辑、删除任务，设置截止日期和优先级，完成任务后勾选关闭。可与流程审批联动（如审批通过后自动生成“后续跟进”待办）。

核心子功能

任务列表：按未完成/已完成分组，支持按截止日期排序。

快速添加：输入任务名称，可选截止时间和优先级（高/中/低）。

提醒通知：利用浏览器 Notification API 在截止时间前提醒。

与审批集成：当差旅、请假等申请被批准后，自动生成“报销申请”或“提交总结”等待办任务。

数据存储：IndexedDB todos 表。

如何符合主题

高频场景：几乎每个职场人每天都会使用。

极致体验：极简交互，一键添加，自动排序，减少认知负担。

逻辑严密：任务可关联业务单据，状态同步准确。

### 2. 公告通知 / 企业动态

职场痛点
公司制度更新、放假安排、活动通知等需要全员知晓，但邮件易被忽略，即时消息易被淹没。

功能描述
管理员端发布公告（标题、内容、附件、置顶），用户端接收公告列表，支持已读/未读状态跟踪，重要公告强制弹窗提醒。

核心子功能

公告发布（管理员）：富文本编辑，可选是否置顶、是否提醒。

公告列表（用户端）：按发布时间倒序，置顶公告优先显示。

已读回执：用户点击查看后标记为已读，管理员可查看每条公告的阅读情况（谁已读/未读）。

离线缓存：公告数据存入 IndexedDB，支持离线阅读。

数据表设计
announcements 表（id, title, content, isTop, isRemind, publishTime, readBy[]）

如何符合主题

高频场景：信息传达是办公基础需求。

极致体验：已读未读一目了然，避免“我没看到通知”的扯皮。

逻辑严密：重要公告强制提醒，确保覆盖率。




### 3.实现建议
数据模型扩展：在 IndexedDB 中增加 todos、announcements 等表。

权限控制：公告发布、通讯录编辑仅限管理员；待办、日历为个人数据。

交互细节：待办任务支持滑动完成（移动端友好）；公告未读数在首页红点提示。
