# 企业办公助手 H5 端功能规格说明书（IndexedDB 本地版）
1. 任务概述
 - 本次POC项目旨在征集一款深度契合职场真实诉求、具备极致用户体验的协同办公应用。参赛者需通过 H5 ，围绕办公高频场景进行产品思维与工程实践的落地。目标是打造一套操作简单、逻辑严密、体验优秀的数字化办公解决方案。
 - 
 - 前端 H5端 分为用户端 和管理端，在登录中提现 参考 README.md 中的技术栈 
 - 后端 node.js 与 Express 架构，数据库 IndexedDB。

## 一、整体架构说明

- **运行环境**：纯 H5 移动端网页，所有数据存储于浏览器 IndexedDB 中。
- **角色划分**：用户端（普通员工）和管理员端，登录时选择角色，操作同一份本地数据。
- **离线可用**：所有功能完全离线运行，无后端依赖（文档预览等外部资源除外）。
- **数据互通**：管理员对数据的修改（如审批、删除会议）会实时反映到用户端界面。
- **初始化**：首次运行时自动创建数据库表及预置账号（admin/123456，user/123456）。

### 数据存储总览

主要数据表及用途：

| 表名 | 用途 |
|------|------|
| users | 用户账号、角色、假期余额 |
| meeting_rooms | 会议室基础信息 |
| reservations | 会议预定记录 |
| trips | 差旅申请记录 |
| leaves | 请假申请记录 |
| attendances | 每日打卡记录 |
| attendance_config | 考勤规则配置 |
| documents | 文档文件及元数据 |
| document_categories | 文档分类 |

## 二、功能模块详述

### 1. 登录模块

**功能描述**  
提供角色选择（用户端/管理员端）和账号密码验证，验证成功后根据角色跳转至对应首页。

**核心逻辑**  
- 角色切换：使用 Tab 或按钮切换用户端/管理员端。
- 账号验证：从 IndexedDB `users` 表中匹配用户名、密码及角色。
- 跳转路由：
  - 用户端 → 会议预定页面
  - 管理员端 → 会议管理页面

**数据表示例（users）**

```json
[
  { "id": 1, "username": "admin", "password": "123456", "role": "admin", "name": "管理员", "annualLeaveBalance": 12, "sickLeaveBalance": 10 },
  { "id": 2, "username": "user", "password": "123456", "role": "user", "name": "张三", "annualLeaveBalance": 5, "sickLeaveBalance": 5 }
]
```

### 2. 会议预定（用户端） / 会议管理（管理员端）
用户端功能

查看会议室列表（名称、容量、设备）。

预定日历：按日期查看每个会议室已被占用的时间段。

新建预定：选择会议室、日期、起止时间（30分钟步长），填写会议主题、参会人，提交前检测冲突。

我的预定：列出当前用户创建的预定，支持取消（仅限未来会议）。

管理员端功能（会议管理页面）

查看所有用户的预定记录，可删除任意预定。

增、删、改会议室信息（名称、容量、设备）。
数据表设计
``` json
// meeting_rooms
[
  { "id": 1, "name": "A会议室", "capacity": 10, "equipment": ["投影","白板"] },
  { "id": 2, "name": "B会议室", "capacity": 4, "equipment": ["电视"] }
]

// reservations
{
  "id": "auto",
  "roomId": 1,
  "userId": 2,
  "subject": "周会",
  "start": "2026-04-12 09:00",
  "end": "2026-04-12 10:00",
  "attendees": [2, 3],
  "createdAt": "2026-04-11T10:00:00Z"
}
```

### 3. 差旅出行
功能描述
用户填写差旅申请单，提交后进入审批状态（待审批→管理员审批）。支持查看历史申请。

核心子功能

新建申请：出差事由、目的地、起止日期、预估交通费/住宿费，可上传附件（图片/文件转 Blob 存储）。

我的差旅：列表展示所有申请，按状态筛选（待审批、已批准、已拒绝）。

撤销申请：仅待审批状态下可撤销。

管理员端功能

查看所有用户的差旅申请，进行审批（通过/拒绝）并填写意见。

统计差旅总费用（简单合计）。

数据表设计
``` json
// trips
{
  "id": "auto",
  "userId": 2,
  "reason": "客户拜访",
  "destination": "上海",
  "startDate": "2026-04-15",
  "endDate": "2026-04-17",
  "estTransportFee": 800,
  "estAccomFee": 1200,
  "attachments": [{ "name": "邀请函.pdf", "blob": "<Blob对象>" }],
  "status": "pending",
  "approvalComment": null,
  "approvedBy": null,
  "approvedAt": null
}
```
### 4. 流程审批
功能描述
统一的审批中心，处理差旅、请假等申请。因只有 user/admin 两个角色，审批流程固定为“发起 → 管理员审批”。

核心子功能

待审批列表（管理员端）：展示所有 status = 'pending' 的申请（差旅、请假等），按类型分组。

审批操作：通过或拒绝，填写意见，更新申请单状态。

已处理列表（管理员端）：查看历史审批记录。

用户端查看进度：在差旅/请假模块中查看自己申请的状态及审批意见。

设计说明

不设多级审批，所有审批由 admin 完成。

审批意见直接写入原申请单的 approvalComment、status 字段。

### 5. 文档查询
功能描述
管理员上传文档，用户端可分类浏览、搜索、在线预览和下载。

核心子功能

文档分类管理（管理员）：预定义分类（公司制度、技术文档等），支持增删分类。

文档上传（管理员）：选择本地文件，将文件以 Blob 形式存入 IndexedDB，记录文件名、大小、分类、标签。

文档列表（用户端）：按分类展示，支持文件名/标签关键字搜索。

预览与下载：点击文档在线预览（PDF 使用 blob URL，Office 文档可尝试转 PDF 或提示下载），支持下载原文件。
数据表设计
```json
// document_categories
[
  { "id": 1, "name": "公司制度" },
  { "id": 2, "name": "技术文档" }
]

// documents
{
  "id": "auto",
  "name": "员工手册.pdf",
  "categoryId": 1,
  "tags": ["人事", "制度"],
  "fileBlob": "<Blob对象>",
  "fileType": "application/pdf",
  "fileSize": 204800,
  "uploadBy": 1,
  "uploadAt": "2026-04-11T10:00:00Z"
}
```
注意事项

限制上传文件大小（建议 ≤20MB）避免超出 IndexedDB 存储限制。

预览 Office 文档需引入第三方库（如 mammoth.js 转 HTML）或引导用户下载后查看。

### 6. 请假打卡
功能描述
用户提交请假申请，记录每日上下班打卡时间。管理员配置考勤规则及假期额度。

请假子功能

请假申请：选择假别（年假、病假、事假）、起止日期、事由，提交后进入审批。

假期余额：展示当前用户各类假期的剩余天数（管理员可手动调整）。

请假记录：查看历史请假及审批状态。

打卡子功能

打卡操作：上班点击“上班打卡”，下班点击“下班打卡”，记录精确时间（同一天仅各一次）。

补卡申请：忘记打卡时提交补卡申请，由管理员审批。

打卡日历：月视图显示每天状态（✓正常、⚠迟到、✗缺卡、请假日）。

迟到规则：管理员配置上班时间（如 09:00），超时打卡记为迟到。

数据表设计
```json
// leaves
{
  "id": "auto",
  "userId": 2,
  "leaveType": "annual",
  "startDate": "2026-04-10",
  "endDate": "2026-04-10",
  "reason": "旅游",
  "status": "pending",
  "approvalComment": null
}

// attendances
{
  "id": "auto",
  "userId": 2,
  "date": "2026-04-11",
  "checkInTime": "09:05",
  "checkOutTime": "18:02",
  "isLate": true,
  "status": "normal"
}

// attendance_config
[
  { "key": "workStartTime", "value": "09:00" },
  { "key": "lateThresholdMinutes", "value": 0 }
]
```
管理员端功能

审批请假申请和补卡申请。

手动调整任意用户的假期余额。

配置考勤规则（上班时间、是否允许补卡等）。

查看全员考勤报表，支持导出 CSV。


### 三、补充说明
1. 数据备份与恢复（可选扩展）
提供“导出数据”按钮，将整个 IndexedDB 导出为 JSON 文件。

提供“导入数据”按钮，从备份文件恢复数据库。

2. 通知机制（可选）
利用浏览器 Notification API 提醒用户审批结果或会议即将开始。

3. 性能与限制
IndexedDB 存储上限一般为 50MB ~ 几百MB（取决于浏览器），建议定期清理或导出。

文档附件单文件建议 ≤20MB，总文档数不宜超过 200 个。

4. 后续升级路径
若需转为客户端-服务器架构，只需将 IndexedDB 操作替换为 API 调用，业务逻辑基本不变。
