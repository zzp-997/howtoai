## 一、首页智能卡片整合

### 1.1 重要公告弹窗

**触发条件：**
- 首次进入首页时自动弹出
- 公告需同时满足：未读 + 重要/置顶 + 未弹窗展示过

**界面说明：**
```
┌─────────────────────────────────────┐
│ 🔔 重要公告                    ✕ │
├─────────────────────────────────────┤
│ [置顶] [制度]                       │
│                                     │
│ 五一放假通知                        │
│                                     │
│ 根据国务院安排，5月1日-5日放假...    │
│                                     │
│ 2026-04-10 09:00                    │
│                                     │
│   [上一条]    [查看详情]    [下一条]  │
└─────────────────────────────────────┘
```

**操作说明：**

| 按钮 | 功能 |
|------|------|
| ✕ 关闭 | 关闭弹窗，标记该公告已展示 |
| 查看详情 | 跳转公告详情页，标记已读 |
| 上一条/下一条 | 多条公告时切换浏览 |

**注意事项：**
- 关闭弹窗后，下次登录不会再弹出相同公告
- 可在「公告通知」页面查看所有公告

## 二、会议秒定功能

### 2.1 智能推荐算法说明

**评分因素：**
```
基础分数 = 3分

加分项：
├── 空闲时间 ≥ 4小时：+1分
├── 空闲时间 ≥ 8小时：+1分（累计+2）
├── 是用户常用会议室：+1分
└── 容量适中（6-12人）：+1分

最高分：5分 ★★★★★
```

**界面示例：**
```
┌─────────────────────────────────────┐
│ 💡 智能推荐                          │
│    为您推荐空闲会议室                │
├─────────────────────────────────────┤
│ ┌─────────┐ ┌─────────┐ ┌─────────┐│
│ │A会议室   │ │B会议室   │ │C会议室   ││
│ │6人       │ │10人      │ │4人      ││
│ │9:00-10:00│ │14:00-16:00│ │全天空闲  ││
│ │空闲      │ │空闲      │ │空闲     ││
│ │★★★★★   │ │★★★★☆   │ │★★★☆☆ ││
│ │  [预定]  │ │  [预定]  │ │  [预定] ││
│ └─────────┘ └─────────┘ └─────────┘│
└─────────────────────────────────────┘
```

---

### 2.2 用户偏好学习

**记录内容：**

| 偏好项 | 说明 |
|------|------|
| 常用会议室 | 最多记录5个，按使用频率排序 |
| 常用时段 | 记录星期几+时间，最多10个 |
| 上次预定信息 | 保存最近一次预定详情 |

**使用偏好示例：**
```
用户张三的预定习惯：
├── 周一 9:00（例会）
├── 周三 14:00（项目评审）
└── 常用 A会议室

→ 系统会在周一/周三优先推荐 A会议室
```

---

### 2.3 操作流程

```
步骤1：进入会议预定
       ↓
步骤2：选择日期（默认今天）
       ↓
步骤3：查看智能推荐卡片
       ├── 点击推荐会议室 → 直接预定
       └── 或浏览全部会议室列表
       ↓
步骤4：填写信息
       ├── 开始/结束时间
       └── 会议主题
       ↓
步骤5：确认预定
       └── 系统自动记录偏好
```

---

### 2.4 冲突检测

**实时检测：**
- 选择时间后自动检测冲突
- 显示已占用时段：「项目周会 09:00-10:00」

**冲突提示：**
```
❌ 该时段已被预定
   项目周会（09:00-10:00）
   请选择其他时段
```

## 四、智能考勤助手

### 4.1 连休推荐算法

**推荐逻辑：**
```
1. 扫描未来90天内的节假日
2. 找出周末/节假日前后的工作日
3. 计算请假天数 vs 连休总天数
4. 按性价比排序（连休天数/请假天数）
5. 过滤超出假期余额的建议
```

**推荐示例：**
```
假设：清明节 4月4日-6日（周四-周六）

推荐方案：
├── 请4月3日（周三）→ 连休4天（周四-周日）
│   性价比：4天/1天 = 4.0
│
├── 请4月7日（周日，调休上班）→ 连休4天
│   性价比：4天/1天 = 4.0
│
└── 请4月3日+4月7日 → 连休5天
    性价比：5天/2天 = 2.5
```

**界面示例：**
```
┌─────────────────────────────────────┐
│ 💡 连休推荐                          │
│    请假效益最大化                    │
├─────────────────────────────────────┤
│ 请假1天，连接清明节连休  → 连休4天  │
│ 04月03日 请假                       │
├─────────────────────────────────────┤
│ 请假1天，连休3天       → 连休3天    │
│ 04月14日 请假（周一）               │
└─────────────────────────────────────┘
```

## 九、常见问题

### Q1: 为什么智能推荐不准确？

**A:** 系统需要积累使用数据。使用越多，推荐越精准。建议多预定会议室、多创建出差申请。

---

### Q2: 连休推荐没有显示？

**A:** 需要满足条件：
- 有年假余额
- 近期有节假日可连接

---

### Q3: 热门搜索为什么是空的？

**A:** 需要全站用户有足够的搜索记录，7天内会逐渐显示。

---

### Q4: 公告弹窗关闭后如何再次查看？

**A:** 进入「公告通知」页面可查看所有公告。

---

### Q5: 用户偏好数据可以重置吗？

**A:** 可以。进入「设置」页面，点击「清除本地数据」即可重置所有偏好数据。

---

### Q6: 费用预估准确吗？

**A:** 费用预估基于预置城市数据，仅供参考。实际费用请根据实际情况填写。
